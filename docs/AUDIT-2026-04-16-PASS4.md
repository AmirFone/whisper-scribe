## Top-of-mind cross-cutting themes

Several findings showed up in multiple agent reports independently. Tackle in this order:

1. **`audio_disk_error` is dead at the presentation layer** (TY-N + CS-N11). Pass-3's AU-NEW-4 wired the `Arc<AtomicBool>` through the backend → `StatusPayload` → `AppStatus.audio_disk_error: boolean` but `StatusBar.tsx` never reads the field. The silent-disk-drop warning the whole change was built to surface is invisible in the UI. One-line `<Show>` + one CSS declaration closes it.

2. **RFC3339 lexicographic comparison in `is_segment_processed`** (P4-S1 + AU4-7 + P4-T1 + N3). `is_segment_processed` uses `last_updated >= ?2` where both sides are `to_rfc3339()` strings. Chrono consistently emits `+00:00` today, but the contract is untested and a `Z`-suffixed or `datetime('now')`-produced row would silently compare differently. Concrete failure mode: false-negatives → re-transcription loop on every restart. Fix: migrate to integer epoch, or pin the format with a `CHECK` constraint and a regression test.

3. **`hour_key` stored in local timezone** (N3 + AU4-7 + P4-S4). `Storage::hour_key_of` uses `Local` tz. A user who changes timezone between runs (travel, `TZ` drift between login shell and `.app` bundle environment, CI running in a non-UTC tz) produces a different `hour_key` for the same `DateTime<Utc>`. `is_segment_processed` misses. Test `test_date_range_query_via_real_api` is latently timezone-dependent — happens to pass on UTC CI runners.

4. **`AppState` god-object is actively spreading** (A4-03 + A4-02 + A4-06 + A4-10). Pass-3's deferral A1 left `pub storage`, `pub transcriber`, `pub is_transcribing` exposed. Post-pass-3 there are now 11 direct field accesses across `pipeline.rs` + `commands.rs`. Three Arc-returning accessor methods (`audio_level_arc`, `segment_started_at_arc`, `audio_disk_error_arc`) leak ownership semantics. Legacy `transcriptions` table still created on every `Storage::new` call with no migration path. `encode_segment_started` / `decode_segment_started` are `pub` for no external reason.

5. **Async IPC response races in the frontend** (Finding 1 + Finding 4 + Finding 7 from wsr4-06 + TY-R). `refreshActiveFilter`, the debounced-search effect, and the `FilterPanel` date-range effect all fire async IPC calls with no cancellation or generation-counter. Out-of-order responses overwrite each other. A single `let fetchGen = 0` counter in `App.tsx` plus `if (gen !== fetchGen) return` guards resolves all three uniformly.

6. **Engine-born-paused produces a filename/content mismatch** (AU4-1). `AudioEngine::new` unconditionally calls `open_new_segment` at construction time. If `pause_flag` is already true (power monitor auto-paused before the engine started), the first callback marks `segment_started_at = UNSET`, leaves the WAV file open but empty, and the eventual audio gets written into a file whose filename timestamp is `T_construct` — which feeds the orphan-dedup path with a wrong capture time. Fix: defer `open_new_segment` to the first unpaused callback.

7. **Python runtime dependencies unpinned + `--break-system-packages` still present** (BS18). BS5 from pass-3 was deferred; it's still the single largest supply-chain hazard. `pip install mlx-whisper` with no version constraint on every first launch. A requirements.txt with pinned versions is a two-line file that closes this.

8. **CI workflow is a skeleton** (BS11 + BS12 + BS13 + BS14 + TT1 + TT8). No `cargo fmt --check`, no `cargo clippy`, no Rust toolchain pin, no npm cache, floating action SHAs, no schedule for the `legacy_schema_tests` gated suite. The workflow that landed in pass-3 is a scaffold; tightening it is mostly one-line YAML additions.

9. **Shutdown hygiene creeping in** (N1 + P4-T4 + A4-07). `shutdown_daemon` holds the daemon mutex across `child.wait()`. `spawn_cleanup_timer` discards its `JoinHandle` and has no stop mechanism. The stderr-forwarder thread's `JoinHandle` is also lost at spawn time. Each is low-impact alone; together they make the eventual C4/AU7 graceful-shutdown work require five retrofit points rather than one cohesive pass.

10. **Testing has structural drift in two more files** (TT5 + TT7). Pass-3 fixed `storage_integration.rs` to test the real `Storage` API; `device_manager_exhaustive.rs` and `audio_and_transcriber.rs` still define local stub functions (`classify`, `parse_timestamp`) instead of importing the production ones. A production-side keyword rename or format change would silently pass.

Beyond the themes, the highest-impact NEW items are:
- **`audio_disk_error` never reset across engine restarts** (AU4-8).
- **`elapsed_seconds` calls `Utc::now()` 46×/sec on the RT thread** (AU4-3). Sample-count pre-gate eliminates 99.9%.
- **`spawn_daemon` misreports crash-on-startup as "Invalid ready JSON"** (P4-T9).
- **`csp: null` + `withGlobalTauri: true` disables all webview security** (BS17).
- **`TOCTOU` between `power_monitor` and `toggle_pause`** (N10) — user's manual pause can be silently overwritten by a system pause landing in the same 15-second ioreg poll window.

# Pass-4 Review: Concurrency, Threading & Lifecycle

(Agent report `/tmp/wsr4-02-concurrency.md`. Severity distribution: 0 high, 3 medium, 7 low. No critical races found.)

### N1. `shutdown_daemon` drops `stdin` then kills — stderr forwarder is not directly unblocked [medium]
The comment claims dropping `stdin` unblocks the stderr forwarder. It doesn't — `stdin` is the *write* end of the child's stdin; the stderr forwarder reads from `ChildStderr`, which closes when the child exits after `kill`. The detached stderr thread can briefly overlap with a new daemon on rapid restart. Fix: store the forwarder `JoinHandle` and join on shutdown.

### N2. `set_pause` / `toggle_pause` Release store ordering — correct today on aarch64 but undocumented [medium]
The `AtomicBool::store` with `Ordering::Release` inside the `pause_state` mutex critical section depends on ARMv8 `STLR` semantics. Add a memory-fence comment so a future port doesn't silently regress.

### N3. `hour_key` is local-tz — timezone change between runs causes orphan dedup misses [medium]
Cross-reference with P4-S4 and AU4-7. Store `hour_key` in UTC.

### N4. Pause-release branch takes `state.lock()` unnecessarily [low]
The `resample_buf.clear()` in the release branch is a no-op (onset already cleared it). Replace with a direct `AtomicI64::store`.

### N5. `is_transcribing = true` stored before transcriber lock — panic leaks the flag [low — deferred A9]
Comment or RAII guard.

### N6. `get_status` elapsed computation unclamped — briefly overflows on rotation boundary [low]
One-tick UI blip when segment rotates. Clamp via `.min(segment_duration_secs as i64)`.

### N7. `spawn_cleanup_timer` holds `PathBuf` by move with no stop mechanism [low]
Future `audio_dir` move/migration would silently operate on stale path.

### N8. `find_orphan_segments` / `list_wavs` doesn't filter by `SEGMENT_PREFIX` [low — deferred AU-NEW-10]
One-line filter addition.

### N9. Pipeline's `is_paused()` TOCTOU — mid-session pause skips the segment permanently [low]
The skip-while-paused path is intentional but loses the WAV forever in-session. Either defer to resume or document the "only recoverable on next launch" semantics.

### N10. TOCTOU between `power_monitor` and `toggle_pause` can silently override user's manual pause [low but real]
Read-modify-write across two lock acquisitions. Fix: collapse into one `set_system_pause_if_unpaused` method with a compare-and-swap inside the mutex.

### P4-S4. `test_date_range_query_via_real_api` fails outside UTC timezone — latent CI flakiness [medium]

### P4-S5. `sanitize_fts_query` produces `"\"-\""` for dash-only tokens — FTS5 can reject [medium — re-raise of deferred S13]

### P4-S7. `SEARCH_HOUR_SLOTS_SQL` hard-caps at 50 with no caller control [low — re-raise of deferred S16]

### P4-S8. `is_segment_processed` silently returns `false` on `prepare_cached` failure [low]
Add a `log::error!` in the `else` branch.

### P4-S9. `wal_autocheckpoint=1000` matches the SQLite default — looks deliberate but isn't [low]
Document intent. Flag connection-scoped PRAGMAs for when S11 (connection pool) lands.

### P4-S10. Missing covering index for `IS_PROCESSED_SQL` [low]

# Pass-4 Review: Frontend State & Effects

(Agent report `/tmp/wsr4-06-frontend-state.md`. Severity distribution: 2 high, 5 medium, 3 low.)

### Finding 1. `TRANSCRIPTION_UPDATED` branch-predicate races across two signal writes [HIGH]
`handleFilterApply` does `setFilterRange(range)` then `setFilterActive(true)`. `clearFilter` reverses. The event listener reads `filterActive()` and separately `refreshActiveFilter` reads `filterRange()`. A race window between the two writes produces a frame where the wrong branch executes. Fix: use `filterRange() !== null` as the sole predicate, or reorder the writes so the dependent read always wins.

### Finding 2. `subscribe_audio_level` Tokio task leaks on unmount [HIGH]
`AudioLevelBars` has no channel close in `onCleanup`. Rust polling task lives until GC eventually closes the channel. Mount/unmount cycles accumulate live tasks. With two simultaneous mounts (deferred F1), ~54 k unnecessary IPC sends per hour.

### Finding 3. `clearFilter` flicker + stale-search trap [medium]
`void loadTimeline()` races a non-empty `searchQuery`. `loadTimeline` early-returns when search is non-empty, leaving stale filter results on screen permanently. Fix: clear search query in `clearFilter` before `loadTimeline`.

### Finding 4. Debounced search races `TRANSCRIPTION_UPDATED` timeline fetch [medium]
Out-of-order IPC responses overwrite each other's `setSlots`. Generation counter resolves.

### Finding 5. `FilterPanel` range-query effect has no visibility guard [medium]
Fires IPC calls when `props.visible` is false, on every signal change (plus pass-3-deferred F2 no-debounce issue).

### Finding 6. `localElapsed` ticker reads `status()` inside a setter updater [medium]
Signal reads inside `setLocalElapsed` setter callbacks are NOT tracked by Solid. Cap is frozen at the first `setInterval` call's value. Move the read into the outer `createEffect` scope.

### Finding 7. `refreshActiveFilter` has no post-await guard [medium]
Simplest of the generation-counter races. `if (results && filterActive() && filterRange() === range) setSlots(results)`.

### Finding 8. `HourSlotCard` scroll-highlight jitter on every keypress [low — re-raise of deferred F9]
Gate `scrollIntoView` on visibility check.

### Finding 9. `tryInvoke<T>` returns `T | null` but callers use falsy guards [low]
Safe for arrays/objects; trap for `tryInvoke<boolean>`. Document the null-check convention and apply it everywhere.

### Finding 10. `unlisten` Promise rejection silently swallowed [low]
Rejected Promise's `.then` fulfillment never fires, so cleanup silently no-ops. Add `.catch`.
