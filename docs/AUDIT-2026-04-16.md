## Part A ‚Äî Findings Ordered by Severity

### CRITICAL ‚Äî do these first, in this order

**C1. `edition = "2024"` in Cargo.toml does not exist**
- Location: `src-tauri/Cargo.toml`
- Fix: change to `edition = "2021"`.
- Risk: none. If this is actually compiling today it's because a nightly toolchain accepts it; pin to 2021 for reproducibility.
- Sanity check before fixing: `grep -n edition src-tauri/Cargo.toml` and `cargo --version`.

**C2. Orphan-segment dedup queries a table that is never written**
- Location: `src-tauri/src/storage.rs:245-254` (`has_transcription_near`), called from `src-tauri/src/lib.rs:307`.
- Problem: `has_transcription_near` queries the legacy `transcriptions` table. All new writes go to `hour_slots`. Result: on every app startup, `has_transcription_near` returns `false` for every orphaned WAV, so orphaned segments are **re-transcribed and re-appended on every restart**, duplicating text inside the matching hour slot and inflating `segment_count`.
- Fix: change the query to target `hour_slots` and match by `start_time` exactly:
  ```rust
  pub fn has_transcription_near(&self, start_time: &str) -> bool {
      let conn = self.conn.lock();
      conn.query_row(
          "SELECT COUNT(*) FROM hour_slots WHERE start_time = ?1",
          params![start_time],
          |row| row.get::<_, i64>(0),
      ).unwrap_or(0) > 0
  }
  ```
  Note: `start_time` on an hour slot records only the FIRST segment's timestamp. Subsequent segments in the same hour do NOT have their individual timestamps stored. This match is therefore weak ‚Äî see F6 for the deeper fix.
- Risk: behavior change on startup; existing users may already have duplicated hour_slots text. Consider a one-shot cleanup on next app launch (or just accept going forward).

Delete the unused `select_best_device`/`list_input_devices` if nothing ever calls them.

  pub const BLUETOOTH_KEYWORDS: &[&str] =

  pub fn is_bluetooth_device(name: &str) -> bool {
      let lower = name.to_lowercase();
      BLUETOOTH_KEYWORDS.iter().any(|kw| lower.contains(kw))

- Risk: low. Both implementations are equivalent today.

**H2. Transcribe+store sequence is copy-pasted in two places**
- Locations: `src-tauri/src/lib.rs:276-340` (`process_orphaned_segments`) and `lib.rs:385-421` (main loop inside `start_pipeline`).
- Fix: extract a single helper. Place it in `lib.rs` or a new `pipeline.rs`:
  ```rust
  fn transcribe_and_store(
      app: &tauri::AppHandle,
      state: &Arc<AppState>,
      path: &std::path::Path,
  ) {
      state.is_transcribing.store(true, Ordering::Relaxed);
      let guard = state.transcriber.lock();
      if let Some(t) = guard.as_ref() {
          match t.transcribe(path) {
              Ok(result) if result.text.is_empty() => {
                  log::info!("Skipping empty/silent segment");
              }
              Ok(result) => {
                  if let Err(e) = state.storage.append_to_hour_slot(
                      &result.text, &result.start_time, &result.device,
                  ) {
                      log::error!("Failed to store transcription: {e}");
                  }
                  app.emit("transcription-updated", &()).ok(); // <- see M3
              }
              Err(e) => log::error!("Transcription failed: {e}"),
          }
      }
      state.is_transcribing.store(false, Ordering::Relaxed);
  }
  ```
- Risk: low; behavior-preserving refactor. Make sure orphan path still passes the audio_dir cleanup afterwards.

**H3. `lib.rs` has 422 lines mixing bootstrap, tray drawing, and the pipeline**

**H5. IPC types duplicated between Rust and TS, drift hazard**

**H6. No `src/api.ts` ‚Äî every `invoke()` is an untyped magic string**
- Locations: `src/App.tsx:47, 56, 62, 69, 76, 80`, `src/components/FilterPanel.tsx:36, 44`.
- Fix: create `src/api.ts` that wraps every Rust command with a typed function. Ideally fed by tauri-specta codegen (H5). If doing manually first:
  ```ts
  // src/api.ts
  import { invoke } from "@tauri-apps/api/core";
  import type { HourSlot, AppStatus } from "./types";

  export const api = {
    getTimeline: (limit: number, offset: number) =>
        invoke<HourSlot[]>("get_timeline", { limit, offset }),
    searchTranscriptions: (query: string) =>
        invoke<HourSlot[]>("search_transcriptions", { query }),
    getStatus: () => invoke<AppStatus>("get_status"),
    togglePause: () => invoke<boolean>("toggle_pause"),
    copyToClipboard: (text: string) =>
        invoke<void>("copy_to_clipboard", { text }),
    getSlotsByDateRange: (fromKey: string, toKey: string) =>
        invoke<HourSlot[]>("get_slots_by_date_range", {
            from_key: fromKey, to_key: toKey,
        }),
    getAvailableDates: () => invoke<string[]>("get_available_dates"),
  };
  ```
- Risk: low; mechanical find-and-replace. Watch for snake_case param names.

**H7. All Rust errors are stringified**
- Locations: every `Result<T, String>` in `commands.rs`, `storage.rs`, `transcriber.rs`.
- Fix: define a shared error enum in `src-tauri/src/error.rs` and return it from commands (so `serde_json` emits a tagged discriminated union):
  ```rust
  #[derive(Debug, thiserror::Error, serde::Serialize)]
  #[serde(tag = "kind", content = "message")]
  pub enum AppError {
      #[error("{0}")] Database(String),
      #[error("{0}")] Transcriber(String),
      #[error("{0}")] Audio(String),
      #[error("{0}")] NotFound(String),
      #[error("{0}")] Validation(String),
      #[error("{0}")] Internal(String),
  }
  impl From<rusqlite::Error> for AppError { /* ... */ }
  ```
  Frontend then gets `{ kind, message }` and can branch.
- Risk: medium. Every command signature changes. Best done after H5 so the TS side gets the enum shape for free.

**H8. Hallucination regex can wipe legitimate repeated phrases**
- Location: `src-tauri/python/mlx_transcribe.py` (the regex list and `clean_hallucinations` ‚Äî auditor #5 flagged patterns that collapse "Yes. Yes. Yes." and similar).
- Fix:
  1. Raise repeat thresholds from "2+" to "3+" for short phrases.
  2. Only apply the "all text is 'thank you' variations" scrub to single short utterances (<= 40 chars).
  3. Log pre/post text at DEBUG when the cleaner fires, so we can A/B compare against a real corpus.
- Risk: medium (behavior change). Test against a captured sample of real transcriptions before merging.

**H9. Transcription can hang forever ‚Äî no timeout**
- Location: `src-tauri/src/transcriber.rs` ‚Äî `daemon.reader.read_line()` blocks indefinitely.
- Fix: add a bounded read-with-timeout (~30s). On timeout, kill the child and drop the `Option<Transcriber>` so the next call re-spawns it. Log a warning event to the frontend.
- Risk: medium. Needs care not to misclassify a slow first-segment (model warm-up) as a timeout. Timeout should be generous (45s) or phased (longer on first call).

**H10. `unsafe impl Send + Sync` on `AppState`**
- Location: `src-tauri/src/lib.rs:26-30`.
- Problem: `rusqlite::Connection` is not `Send` on all platforms. Today the `Mutex` makes it correct in practice, but the `unsafe impl` hides this from the compiler.
- Fix: remove the `unsafe impl` and restructure so safety is derived. Simplest path: use `r2d2` + `r2d2_sqlite` (adds 2 deps, removes `unsafe`). Or: move storage behind a channel-served worker thread (no extra deps, bigger refactor).
- Risk: medium; behavior-preserving if r2d2 path chosen.

**H11. `eprintln!` in production audio callback**
- Locations: `src-tauri/src/audio_engine.rs:156` and `:214`.
- Fix: remove or gate behind `cfg!(debug_assertions)` or `log::debug!`. The hardcoded `count == 0 || count == 100 || count == 1000` also burns a branch on every callback ‚Äî wrap it:
  ```rust
  #[cfg(debug_assertions)]
  if count < 3 || count % 1000 == 0 {
      log::debug!("[AUDIO] callback #{count}: ...");
  }
  ```
- Risk: none.

**M1. Pause vs system-pause state can silently overwrite user intent**
- Location: `src-tauri/src/power_monitor.rs:30-39` interacting with `lib.rs:20-21` and `commands.rs:toggle_pause`.
- Problem: if user manually pauses, then the screen locks (setting `paused_by_system=true`) and unlocks, the monitor auto-resumes regardless of user intent.
- Fix: replace the pair of booleans with a single `Mutex<PauseReason>`:
  ```rust
  #[derive(Debug, Clone, Copy, PartialEq)]
  pub enum PauseReason { None, Manual, System }
  ```

- Risk: low. Add a test for the "manual pause survives lock/unlock" scenario.

**M4. `append_to_hour_slot` is a read-then-write**
- Location: `src-tauri/src/storage.rs:99-127`.
- Fix: collapse to a single atomic statement:
  ```sql
  INSERT INTO hour_slots (hour_key, text, start_time, last_updated, device, segment_count)
  VALUES (?1, ?2, ?3, ?3, ?4, 1)
  ON CONFLICT(hour_key) DO UPDATE SET
      text = text || ' ' || excluded.text,
      last_updated = excluded.last_updated,
      device = excluded.device,
      segment_count = segment_count + 1;
  ```
  Note that `last_insert_rowid()` after an `ON CONFLICT DO UPDATE` returns 0 on the update path. The only caller ignores it today, but verify.
- Risk: low.

**M5. Device name overwritten on every segment append**
