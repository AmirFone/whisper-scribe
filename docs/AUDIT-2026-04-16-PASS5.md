## Cross-cutting themes

### Theme 1 — `subscribe_audio_level` / `AudioLevelBars` lifecycle (4 agents)

**Agents:** Architecture (A5-03), Concurrency (F1), Frontend-state (F5-2), Frontend-types (F5-TY-09).

The component mounts in two places (`StatusBar` always, `Timeline` conditionally); each mount opens an independent Tokio polling task with no cancellation. The `Channel` is never closed in `onCleanup`. Tasks accumulate over the session. This has been the single most-flagged item across all five passes (C2/C9/A3 in pass-3, Finding 2 in pass-4, now four agents in pass-5). The stopgap fix (abort handle on `AppState`) is ~5 lines; the strategic fix (push events via `Emitter::emit`) has been deferred since pass-1.

**Status:** Known deferred. Not a convergence blocker.

### Theme 2 — Shutdown hygiene (3 agents)

**Agents:** Concurrency (F3, F9, F10), Transcriber (T5-01), Architecture (implicit).

Pipeline thread handle discarded, power monitor never joined, cleanup timer has no stop flag, `shutdown_daemon` holds mutex across `child.wait()`. Each is low-impact alone; together they make the eventual graceful-shutdown pass require five retrofit points. This was explicitly deferred as "needs its own pass" in passes 3 and 4.

**Status:** Known deferred. Not a convergence blocker.

### Theme 3 — Dead code: `count_samples` / `end_time` / `confidence` (2 agents)

**Agents:** Architecture (A5-02), Transcriber (T5-02).

`count_samples` opens the WAV file solely to populate `end_time`, a `dead_code` field with zero consumers. The `dead_code` compiler warning has survived all five passes. Deletion is a two-line change.

**Status:** Known deferred (A4-09 / P4-T2). Not a convergence blocker.

### Theme 4 — `localElapsed` cap frozen in setter callback (3 agents)

**Agents:** Architecture (A5-05), Frontend-state (F5-5), Frontend-types (F5-TY-10).

`status().segment_duration_secs` read inside a `setLocalElapsed` setter callback is not tracked by Solid. The cap is frozen at the value from the first interval call. This is a genuine reactivity bug but low-impact: `segment_duration_secs` is effectively a compile-time constant (env var, read once). Finding 6 from pass-4.

**Status:** Known deferred. Low impact.

### Theme 5 — FilterPanel visibility guard + stale snapshot (2 agents)

**Agents:** Architecture (A5-04), Frontend-state (F5-4, F5-10).

Range-query effect fires when panel is invisible (deferred F2/Finding 5). New: `filteredSlots()` snapshot passed on Apply can be stale if user changes selectors while IPC is in-flight — the `panelFetchGen` discards the response but doesn't clear stale results. This is a new edge case introduced (or at least exposed) by the pass-4 generation counter work.

**Status:** Partially new. Low impact — requires rapid selector changes concurrent with slow IPC.
